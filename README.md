# p1-w1-d1-t4 · Функция-конструктор: секундомеры

> Тип: объёмная (new + prototype, до классов) · Репозиторий: один отдельный

> Подключить `.js` к HTML через `<script>`, отсмотреть в браузере (или запустить в консоли).

## Задача

До `class` объекты одного «типа» создавали функцией-конструктором: поля клали в `this`, общие методы — в `.prototype`. Понять это важно, потому что `class` — просто обёртка над этим. Собираешь руками конструктор секундомеров: несколько независимых счётчиков времени, у каждого своё состояние, но общие методы.

```
функция-конструктор (как классы до 2015)
┌──────────────────────────────────────────────────────┐
│ function Timer(label){ this.label=...; this.ms=0 }   │
│ методы в Timer.prototype — общие для всех            │
│ new Timer('work') → свой объект со своим состоянием  │
│ доказать: метод в prototype, new привязал цепочку    │
└──────────────────────────────────────────────────────┘
new + prototype настраивают прототипную цепочку
```

## Дано

```js
// тебе написать функцию-конструктор Stopwatch по описанию
// проверочные ярлыки:
const labels = ["Работа", "Отдых", "Учёба"];
```

## Что нужно сделать (по релизам)

**Release 1 — конструктор с состоянием.** Написать функцию-конструктор `Stopwatch(label)`: кладёт в `this` поле `label` и `seconds = 0`. Создать через `new` несколько секундомеров (можно из `labels`). Показать, что у каждого своё независимое состояние.

**Release 2 — методы в prototype.** Добавить в `Stopwatch.prototype` методы: `tick()` (увеличивает `seconds` на 1, возвращает `this` — чтобы можно было чейнить), `reset()` (обнуляет), `format()` (возвращает «Работа: 00:05» — минуты:секунды с ведущими нулями). Показать, что методы общие для всех секундомеров, а `this` внутри — конкретный экземпляр. Продемонстрировать чейнинг: `sw.tick().tick().tick().format()`.

**Release 3 — доказать механику.** Показать, что `tick` лежит в прототипе (`sw.hasOwnProperty("tick")` → `false`, но метод работает), что `new` привязал прототип (`Object.getPrototypeOf(sw) === Stopwatch.prototype`), и что два секундомера делят один и тот же экземпляр метода (`sw1.tick === sw2.tick` → `true`). В `NOTES.md` по шагам расписать, что делает `new`.

## Ожидаемый результат

- Новый секундомер имеет своё независимое состояние; после нескольких «тиков» формат показывает накопленное время как `мм:сс` с ведущими нулями, а сброс возвращает его к нулю.
- Тики можно чейнить (метод возвращает сам объект).
- Проверки показывают: метод не является собственным свойством экземпляра (он в прототипе), а два разных секундомера делят один и тот же экземпляр метода.

## Критерии приёмки

* Конструктор кладёт состояние в `this`; экземпляры независимы; методы в `.prototype` общие.
* `format` корректно форматирует минуты:секунды с ведущими нулями; чейнинг через возврат `this` работает.
* Доказано, что метод в прототипе и общий для экземпляров; в `NOTES.md` по шагам описан `new`.
